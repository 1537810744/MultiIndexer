# MultiIndexer 项目启动指南

## 前置条件

### 1. Go 1.22+
```bash
go version
# 期望: go version go1.22.x windows/amd64
```
如未安装：https://go.dev/dl/ 下载安装。

### 2. Docker Desktop
```bash
docker --version
docker compose version
```
如未安装：https://www.docker.com/products/docker-desktop/ 下载安装，启动后等右下角图标变绿。

## 启动步骤

### 第一步：启动 Docker 中间件

```bash
cd C:\Users\admin\Desktop\MultiIndexer
docker compose up -d
```

等待所有容器 healthy（首次约 2-5 分钟）：

```bash
docker compose ps
```

期望看到三个容器的 STATUS 都是 **healthy**。

### 第二步：启动 Listener（打开一个新终端）

```bash
cd C:\Users\admin\Desktop\MultiIndexer\listener
go run .
```

Mock 模式（默认）不需要 RPC 节点，程序自动生成模拟数据。

### 第三步：启动 Processor（打开第二个终端）

```bash
cd C:\Users\admin\Desktop\MultiIndexer\processor
go run .
```

### 第四步：启动 Fake API（打开第三个终端）

```bash
cd C:\Users\admin\Desktop\MultiIndexer\fake-api
go run .
```

### 第五步：打开浏览器访问

```
http://localhost:9090
```

## 凭证说明（无需手动设置）

Go 代码中的默认值已匹配 docker-compose.yml，**不需要设置环境变量**：

| 服务 | 地址 | 用户名/密码 |
|------|------|-------------|
| RabbitMQ | `localhost:5672` | `admin` / `admin` |
| MySQL | `localhost:3307` | `indexer` / `indexerpass` |
| Redis | `localhost:6379` | 无密码 |

> 注意：docker-compose.yml 中 RabbitMQ 用户是 **admin:admin**（非 guest:guest）。

管理后台：
- RabbitMQ 管理界面：http://localhost:15672 （admin/admin 登录）
- MySQL：`mysql -h 127.0.0.1 -P 3307 -u indexer -pindexerpass indexer`

## 验证系统

1. 浏览器打开 http://localhost:9090 — 应看到仪表板
2. http://localhost:9090/api/health — 健康检查（mysql=true, redis=true）
3. http://localhost:9090/api/events — 最近事件列表
4. http://localhost:9090/api/stats — 事件/告警统计

三个终端窗口应该都在持续输出日志，表示数据在实时流动。

## 关闭系统

每个终端按 **Ctrl+C** 停止 Go 程序，然后：

```bash
docker compose down
```

## 常见问题

### Q: RabbitMQ 报 `PLAIN login refused: user 'guest'`
说明代码中的默认值仍是 `guest:guest`。更新代码到最新版本（默认已是 `admin:admin`）。

### Q: Processor 报 MySQL 连接失败
确认 Docker 容器在运行：`docker compose ps`。确认端口是 **3307**（不是 3306）。

### Q: Redis 密码是什么？
本项目 Redis 没设密码。docker-compose.yml 中 Redis 服务没有配置密码。

### Q: go run 报 missing go.sum
```bash
go mod tidy
```
